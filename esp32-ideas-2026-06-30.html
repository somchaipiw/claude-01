🚗 Automotive IoT
ESP32 TWAI™ — Built-in CAN Bus Controller สำหรับ Automotive IoT
ESP32 มี CAN controller ในตัวชื่อ TWAI (Two-Wire Automotive Interface) รองรับ CAN 2.0B ทำให้สามารถต่อรถยนต์ผ่าน OBD-II ได้โดยตรงด้วยแค่ IC transceiver เพิ่ม เช่น MCP2551 หรือ TJA1050 ราคาต่ำกว่า 30 บาท — เปิดประตูสู่โปรเจกต์ยานยนต์ราคาประหยัด
CAN 2.0B ISO 11898 OBD-II TWAI Controller 1 Mbps TJA1050 / MCP2551
ทำอะไรได้บ้าง
อ่านค่า RPM / Speed / Coolant Temp, ดึง Fault Code (DTC), สร้าง Dashboard ไร้สาย, บันทึก Trip Log, ส่งข้อมูลไป MQTT/Cloud
// ตั้งค่า TWAI driver #include "driver/twai.h" twai_general_config_t g_config = TWAI_GENERAL_CONFIG_DEFAULT(GPIO_NUM_21, GPIO_NUM_22, TWAI_MODE_NORMAL); twai_timing_config_t t_config = TWAI_TIMING_CONFIG_500KBITS(); twai_filter_config_t f_config = TWAI_FILTER_CONFIG_ACCEPT_ALL(); twai_driver_install(&g_config, &t_config, &f_config); twai_start();

🔗 Matter over Thread

🚗 Automotive
ESP32 OBD2 Vehicle Telematics — ดีไวซ์วินิจฉัยรถยนต์ DIY
สร้างอุปกรณ์วินิจฉัยรถยนต์แบบ WiFi ด้วย ESP32 + TJA1050 — ดึงข้อมูล Engine RPM, ความเร็ว, อุณหภูมิ, อ่าน DTC (Fault Code) แล้วส่งไปแสดงบน Web Dashboard หรือ MQTT broker แบบ Real-time
1
Hardware: ESP32 + TJA1050 transceiver + OBD-II connector (DB9 หรือ socket) + 12V→5V converter
2
Library: ใช้ OBD2_CAN_Bus_Library หรือ esp32_obd2 บน GitHub ของ MagnusThome
3
Software: สร้าง Web server บน ESP32 แสดง Dashboard + MQTT publish ทุก 1 วินาที
4
Advanced: เพิ่ม GPS (NEO-6M) + SD Card Logger สำหรับบันทึก Trip ทั้งหมด
// อ่านค่า RPM จาก OBD2 OBD2 obd; float rpm = obd.pidRead(ENGINE_RPM); float speed = obd.pidRead(VEHICLE_SPEED); float coolant = obd.pidRead(ENGINE_COOLANT_TEMP); mqtt.publish("car/rpm", String(rpm).c_str());

👃 Electronic Nose
ESP32 + BME688 eNose — จมูกอิเล็กทรอนิกส์ที่ฝึก AI ได้
สร้าง Electronic Nose ด้วย BME688 + ESP32 โดยใช้ Bosch BME AI-Studio ฝึกโมเดล Machine Learning จำแนกกลิ่นได้โดยไม่ต้องเขียนโค้ด ML — เหมาะสำหรับตรวจจับกาแฟเสีย, อาหารเน่า, แก๊สรั่ว, หรือควันไฟ
1
เก็บ Dataset: วาง BME688 ใกล้กลิ่นแต่ละประเภท เก็บข้อมูล Resistance Pattern 30+ ตัวอย่าง/กลิ่น
2
ฝึกโมเดล: ใช้ Bosch BME AI-Studio หรือ scikit-learn (Random Forest / SVM) จำแนก Pattern
3
Deploy: Export โมเดลเป็น C array ฝังลงบน ESP32 ด้วย ESP-DL หรือ TFLite Micro
4
Alert: MQTT alert เมื่อตรวจจับกลิ่นผิดปกติ + บันทึก timestamp ลง SD
ผลงานวิจัยล่าสุด 2026
งานวิจัยใช้ BME688 x8 บอร์ด + HUZZAH32 ESP32 จำแนก Coffee Powder 3 ชนิดจาก 3,583 ตัวอย่างได้ความแม่นยำ >95% ด้วย Random Forest

📡 MQTT 5.0
ESP32_MQTTv5 — MQTT Protocol รุ่นใหม่ที่ IoT Enterprise ต้องการ
MQTT 5.0 นำเสนอ feature ใหม่ที่ขาดไม่ได้สำหรับ Enterprise IoT: Message Expiry, Topic Alias (ลด bandwidth), Shared Subscriptions (load balance), Enhanced Authentication และ Reason Codes ที่ละเอียดกว่า
MQTT 5.0 QoS 0/1/2 Topic Alias Shared Subscription Message Expiry Will Delay
MQTT 5.0 vs 3.1.1 — ความแตกต่างสำคัญ
• Topic Alias: ย่อ topic ยาวเป็น ID ตัวเลข ลด payload 60-80%
• Session Expiry: กำหนดอายุ session ได้ละเอียด
• Reason Codes: debug ปัญหาได้ง่ายกว่าเดิมมาก
