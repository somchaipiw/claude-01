ESP32-H2 — Zigbee / Thread / Matter Gateway
ชิปเฉพาะทางสำหรับ Smart Home Protocol

ESP32-H2 เป็นชิป RISC-V single-core 96 MHz ที่ไม่มี Wi-Fi แต่รองรับ IEEE 802.15.4 (Zigbee & Thread) พร้อม BLE 5.0 ได้รับการรับรอง Thread 1.3.0 และ Zigbee 3.x อย่างเป็นทางการจาก Connectivity Standards Alliance

เหมาะสำหรับ: อุปกรณ์ sensor ประหยัดพลังงาน, end-node ใน Zigbee network, Matter end-device ที่ต้องการแบตเตอรี่ยาวนาน

CPURISC-V 32-bit, 96 MHz
RAM320 KB SRAM
โปรโตคอลZigbee 3.x, Thread 1.3, BLE 5.0
Flash4 MB (module)
ราคาโมดูล~$2-3 USD
💡 เคล็ดลับ: จับคู่ ESP32-H2 (Zigbee/Thread) + ESP32-C6 (Wi-Fi 6) เพื่อสร้าง Thread Border Router หรือ Matter-Zigbee Bridge โดยไม่ต้องซื้อ hub แยก
Zigbee 3.x Thread 1.3 Matter BLE 5.0 Low Power

ESP32-S3 — USB OTG + AI + Deep Sleep Ultimate Guide
Dual-core AI Accelerator สำหรับ Edge Intelligence

ESP32-S3 คือตัวเลือกสูงสุดสำหรับ Edge AI ในระดับ Sub-$5 รองรับ USB OTG (Host + Device), dual-core 240 MHz พร้อม 8 MB PSRAM ในรุ่น N16R8 และมี ULP coprocessor ทั้ง RISC-V และ FSM

Deep Sleep Power Profile: ในโหมด Deep Sleep กินกระแสเพียง ~10 µA; ULP mode กิน ~100 µA เหมาะสำหรับ sensor node อ่านค่าทุก 5-15 นาที

Active Mode (240 MHz + Wi-Fi): ~240 mA
Modem Sleep: ~20 mA
ULP (RISC-V) Active: ~100 µA
Deep Sleep: ~10 µA

ESP-BLE-MESH: Self-Healing Network สำหรับ Industrial IoT
ESP-IDF v5.x — Many-to-Many Communication

ESP-BLE-MESH ช่วยให้ ESP32 หลายตัวสร้างเครือข่ายแบบ mesh ที่ข้อความส่งผ่านหลาย hop อัตโนมัติ ขยายพื้นที่ครอบคลุมโดยไม่ต้องการ coordinator hardware แพงๆ ต่างจาก Zigbee

สถาปัตยกรรม: Node แต่ละตัวทำหน้าที่ทั้ง Relay, Proxy และ Friend Node ได้ รองรับถึง หลายพัน node ต่อ network

  • Provisioner node กำหนด network key และ address ให้ทุก node
  • Node ทุกตัว relay ข้อความต่อโดยอัตโนมัติ (flooding หรือ directed)
  • หากเส้นทางล้มเหลว เครือข่ายหา path ใหม่ได้เอง
  • Low-power node ใช้ Friend node เก็บข้อความแทนในช่วง sleep
📌 ข้อได้เปรียบ vs Zigbee: ไม่ต้องซื้อ Zigbee coordinator, ใช้ BLE ที่มีอยู่ใน ESP32 ทุกรุ่น, integrate กับ Wi-Fi cloud gateway ได้โดยตรง

4-20mA Industrial Sensor Interface กับ ESP32
มาตรฐานอุตสาหกรรม — Noise-Immune สำหรับโรงงาน

มาตรฐาน 4-20mA current loop เป็น standard ของโลกอุตสาหกรรมเพราะทนทานต่อ noise ทางไฟฟ้าสูง และส่งสัญญาณได้ระยะไกลหลายร้อยเมตร

วิธีเชื่อมต่อกับ ESP32 ทำได้ง่ายมาก: ใช้ resistor 250Ω แปลง 4-20mA → 1-5V แล้วต่อเข้า ADC ของ ESP32 (หรือใช้ INA219 สำหรับความแม่นยำสูง)

// อ่านค่า sensor 4-20mA const int ADC_PIN = 34; float readCurrent4_20mA() { int raw = analogRead(ADC_PIN); // 0-4095 float voltage = raw * 3.3f / 4095.0f; float mA = (voltage / 250.0f) * 1000.0f; // R=250Ω float percent = (mA - 4.0f) / 16.0f * 100.0f; return percent; // 0-100% }
